    Practical Machine Learning

    Welcome to Practical Machine Learning, a code-first course to learn the practical side of Machine Learning. Here you will learn how to build and run your own Machine Learning models in python. More importantly you will learn practical tips and tricks in Machine Learning and Software Development, which traditionally comes through years and years of experience, and so, this course offers you an opportunity to grow much faster. Every session is designed to help you build state-of-the-art models, and learn how to tackle real-life situations like industrial experts.

    What’s unique about this course?

    There are great resources out there to learn concepts in Machine Learning, there arenâ€™t many resources to learn how to code. This mini-course teaches cocepts in Machine Learning through code. No unnecessary theory and little to no math needed. No advanced coding skills required. This is a great resource to learn how to build your own models in python, and how to use already existing libraries to the best of your advantage!

    Who is this course for?

    There are no prerequisites to this course as such. But it is reccommended to have a basic understanding of a concept before learning how to implement it through code. But anyone who decides to start their journies in Machine Learning through this course are more than welcome to do so. The only other requirement in that case is a commitment to put in extra effort.

    How should I go about this course?

    The best way to go about this course is to do this in parallel with another theory inclusive course in Machine Learning. It may be a course in your university, or an online MOOC. Once you have a basic understanding of the concept (meaning, you have an intuition about how it works, and know the working (without the math and the overly technical details), come back here and pick the corresponding session. Run all the code sequentially (Type them, don’t copy paste!), and try solving the exercise provided at the last. Once you do that, its recommended to go through the entire session again, and this time try writing all the code on your own, without looking. If you get stuck, its okay to take a peek into the original code, and then continue writing without looking. The idea is to understand the steps involved, and to gain the confidence to code independently.

    What can I expect to learn from this course?

    The focus of this course is to learn how to build your own Machine Learning models, and solve different kinds of problems through those models. So, besides learning how to implement the code for the model, you will learn how to pick up a problem of any complexity, tackling each challenge that comes along the way, and formatting the problem in a way that can be solved by your Machine Learning model. This is going to give you the confidence to face the challenges that you will face when you come across real life problems. Another focus of this course is to gain good coding skills. Throughout this course, we will focus on good coding practices, like using advanced python tools, using library APIs in a structural manner, refactoring code, and working in an object oriented programming manner. Finally, you will learn the State-of-the-Art tools and techniques used in the industry, and beyond!

    Setting up Cloud based Jupyter Notebooks

    You can set up any cloud based system that supports Jupyter Notebooks, like GCP, PaperSpace, Kaggle, Google Colab, etc.

    In this course, we reccommend Google Colab. Google Colaboratory (called Colab for short) is a jupyter environment from Google, and easily integrates with all other google services. For example, you can easily integrate it with Google Drive, or share the notebook with other people. Google Colab is a convenient way to run ipynb notebooks with minimal setup. You don’t need to install most commonly-used python libraries. You need to have a google account for this.

    Setting up Google Colab

    The simplest and most organised way to set up a Colab notebook is to login to your Google Drive. You can optionally make a folder at a suitable location if you wish to organise all your Colab Notebooks in a specific folder. Once you navigate to a folder of your choice, simply create a new colab notebook, by clicking on New –> More –>Google Colaboratory. This will open a new colab notebook on the website colab.research.google.com.

    Give your notebook a suitable name by clicking on the title at the top (for example, 01 Basics of Python and ML.ipynb)(Note that jupyter notebooks have an extension of .ipynb). The instructions to use this notebook are described in 01 Basics of Python and Introduction to Machine Learning. Whenever you wish to run a piece of code, you simply type the code in a cell and run it.

    The notebook is auto-saved, so you don’t have to do it manually. Whenever you wish to return to the notebook, simply go back to your Google Drive directory, and double click on the Notebook, and then click on Open With –> Google Colaboratory.

    Setting up Jupyter Notebooks on your local system

    You can find a detailed, step-by-step guide here. Once you open a Jupyter Notebook, the same instructions that are described above in the Colab section are followed! However, you might need to install libraries whenever you use them for the first time. You can do this by following this guide.

    Cloning the Github Repo

    Though there is no substitute to typing your own code, there is a minor setback - You won’t get the descriptions, pictures, gifs, etc that you see on the blog (Jupyter Notebooks allow you to do that!), unless you explicitly type everything manually. If you feel that you need all the pictures and descriptions along with the code for better understanding, or for future references, you can use pre-written .ipynb file, that are present in the GitHub Repo. If you use a terminal like the Windows Command Line, Linux Based Bash system, or Mac Terminal, you can clone the repo on your local system and upload the notebooks to your google drive to a preferred directory. In case you don’t use a terminal, downloading a zip folder of the repository directly would be a better option. You can simply do so by going to the GitHub reposi*tory, click on the green button that says Code, and click on download zip. Unzip the downloaded folder and upload the .ipynb files to your Google Drive (if you want to use Google Colab), or directly open the .ipynb files using the jupyter system installed on your local system.
